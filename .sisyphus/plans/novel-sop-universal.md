# 通用小說生成標準化流程手冊 (Universal Novel Generation SOP)

> **版本**: v7.1 - 自動連續生成版本
> **建立日期**: 2026-04-06
> **適用範圍**: 任何長度、任何類型、任何語言的小說生成任務
> **默認模式**: 除非用戶要求，否則自動連續生成全部章節

---

## 一、核心設計原則

### 1.1 設計目標
- **通用性**: 適用於任何故事大綱（簡單或詳細）
- **適應性**: 自動調整生成策略根據輸入複雜度
- **連貫性**: 確保章節間劇情邏輯連貫
- **質量保障**: 內建品質檢查機制
- **自動連續**: 除非用戶明確要求，否則默認自動生成全部章節

### 1.2 生成模式

```markdown
## 默認行為

### 自動模式（默認）
- 用戶提供大綱 → 確認大綱 → 自動生成全部章節
- 不需要中途確認
- 直到全部完成或用戶主動停止

### 確認模式（用戶要求時）
- 用戶說明「每X章確認一次」
- 僅在指定節點暫停確認
- 確認後繼續自動生成

### 用戶要求格式
- 「每10章確認一次」
- 「生成5章後停下來問我」
- 「分批次生成」
```

### 1.3 輸入複雜度分類

| 等級 | 描述 | 範例 | 生成策略 |
|------|------|------|----------|
| **Level 1** | 極簡大綱 | "一個關於復仇的故事" | 需要大量填充 |
| **Level 2** | 簡單大綱 | "高中生發現超能力，與反派戰鬥" | 部分填充 |
| **Level 3** | 中等大綱 | "主角名：Alex，身份：學生，能力：時間暫停，反派：黑暗組織，目標：拯救世界" | 少量填充 |
| **Level 4** | 詳細大綱 | 包含角色詳細設定、世界觀、章節大綱、懸念設計 | 最小填充 |

## 二、標準化流程 (SOP)

### Phase 1: 輸入分析與規劃

#### Step 1.1: 解析用戶輸入

```markdown
## 用戶輸入分析表

### 基本資訊提取
- 故事類型: [類型]
- 主要元素: [元素1, 元素2, ...]
- 目標長度: [章節數/字數]
- 語言: [語言]

### 角色提取
- 主角: [名字/描述]
- 反派: [名字/描述]
- 配角: [列出]

### 世界觀提取
- 場景: [地點/世界]
- 規則: [特殊規則]
- 背景: [時代/設定]

### 情節提取
- 起始點: [開頭]
- 目標/願望: [目標]
- 障礙: [障礙]
- 高潮: [高潮設計]
```

#### Step 1.2: 識別缺少的資訊

**缺失識別清單**:
- [ ] 主角姓名
- [ ] 主角背景/動機
- [ ] 反派設定
- [ ] 世界觀細節
- [ ] 故事類型/風格
- [ ] 目標長度
- [ ] 結局方向

**處理策略**:
- Level 1-2: 啟動[自動填充模組]
- Level 3-4: 使用用戶提供的資訊

#### Step 1.3: 生成結構規劃

**根據目標長度選擇**:

| 目標長度 | 結構規劃 |
|----------|----------|
| 短篇 (1-5章) | 線性結構：開端→發展→高潮→結尾 |
| 中篇 (6-15章) | 三幕結構 + 若干支線 |
| 長篇 (16-50章) | 多卷結構 + 捲節點 + 主支線交織 |
| 超長篇 (50+章) | 史詩結構 + 多重高潮 + 種族命運 |

### Phase 2: 世界觀與角色建立

#### 2.1 自動填充模組 (當用戶大綱不足時)

**角色生成器**:

```markdown
## 角色模板

### 主角填充
[如果缺失姓名]
- 根據故事類型生成合適名字
- 例：科幻→Alex/Maya, 奇幻→Aria/Kael, 都市→小雨/阿強

[如果缺失動機]
- 根據目標反推合理動機
- 例：拯救世界→保護所愛/責任感/贖罪

### 反派填充
- 必須有與主角對立的根本原因
- 最好有悲劇背景或扭曲理念
- 避免「純邪惡」設定

### 配角填充
- 導師型: 提供知識/工具
- 盟友型: 提供幫助/情感支持
- 犧牲型: 推動主角成長
```

**世界觀生成器**:

```markdown
## 世界觀模板

### 場景設定
- 如果是現實世界：添加真實地點/時代背景
- 如果是幻想世界：設計獨特規則 + 視覺特色
- 如果是科幻世界：設計科技水平 + 社會結構

### 核心規則 (必須清晰)
- 這個世界最重要的規則是什麼？
- 角色如何運用這個規則？
- 打破規則會有什麼後果？

### 視覺/氛圍參考
- 用3個關鍵詞描述整體風格
- 例：黑暗/懸疑/浪漫 / 明亮/熱血/成長
```

#### 2.2 世界觀一致性檢查

```markdown
## 一致性檢查清單

- [ ] 規則是否在全書中一致適用？
- [ ] 角色能力是否有合理限制？
- [ ] 場景描寫是否符合世界觀？
- [ ] 科技/魔法水平是否一致？
```

### Phase 3: 情節擴展引擎

#### 3.1 情節擴展原則

**衝突層次**:
1. **外在衝突**: 與環境/敵人/社會的衝突
2. **內在衝突**: 角色內心的掙扎/恐懼/欲望
3. **人際衝突**: 角色之間的誤會/背叛/愛恨

**每章必備元素**:
- 至少一個衝突（大小均可）
- 角色決定影響劇情
- 為下一章留懸念

#### 3.2 懸念管理

```markdown
## 懸念設計模板

### 悬念類型
1. 資訊懸念: 隱瞞關鍵資訊讓讀者好奇
2. 命運懸念: 角色未來會如何
3. 謎題懸念: 等待揭開的秘密
4. 情感懸念: 角色關係的發展

### 懸念層次
- 章內懸念: 本章結束時揭曉
- 捲內懸念: 本卷結束時揭曉
- 終極懸念: 全書結尾揭曉

### 懸念平衡
- 每3章至少1個主要懸念
- 每卷至少1個謎團
- 開頭設置終極懸念
```

### Phase 4: 章節生成

#### 4.1 生成前準備

```markdown

1. [ ] 讀取上一章結尾的[狀態記錄]
2. [ ] 確認本章目標（情節推進/角色揭示/世界觀擴展）
3. [ ] 確認懸念安排（本章懸念 + 預埋下一章）
4. [ ] 確認角色狀態（情緒/位置/關係）
```

#### 4.2 通用章節模板

```markdown
# 第 [N] 章：[標題]

[時間/地點]: [時間描述] [地點描述]

---

[場景開始 - 銜接上一章]

[場景發展 - 按以下比例]

## 內容比例參考 (可根據類型調整)

### 選項A: 偏重動作冒險類
- 場景/動作: 40%
- 對話: 30%
- 心理: 20%
- 描述/氛圍: 10%

### 選項B: 偏重情感心理類
- 心理/獨白: 40%
- 對話/互動: 30%
- 場景/氛圍: 20%
- 動作/情節: 10%

### 選項C: 偏重懸疑推理類
- 線索/謎題: 35%
- 對話/審訊: 30%
- 場景/細節: 20%
- 心理推理: 15%

---

[本章結尾 - 懸念設計]

---

> **全局狀態記錄（每章必備）:**
> * 當前時間線: [時間進度]
> * 角色狀態: [心理/位置/關鍵物品]
> * 情節進度: [本章完成度]
> * 懸念追蹤: [進行中/待揭曉/已完成]
> * 伏筆鋪設: [本 章埋下的伏筆]
```

## 三、特殊情況處理

### 3.1 用戶只提供極簡大綱時

**處理流程**:

1. **請求澄清**: 至少問3個關鍵問題
   - 故事類型偏好？
   - 主角想要什麼？
   - 反派是誰？
   
2. **自動假設**: 對次要元素進行合理假設
   - 名字: 使用常見名字
   - 背景: 使用通用設定
   - 世界: 基於類型的典型設定

3. **明確告知**: 告訴用戶哪些是假設，可以修改

### 3.2 用戶要求中途修改方向

**處理流程**:

1. **評估影響**: 改變會影響多少章節？
2. **更新規劃**: 調整後續章節大綱
3. **修訂記錄**: 更新全局狀態記錄
4. **向後兼容**: 盡量讓新情節與已有情節兼容

### 3.3 卡文/沒有靈感時

**處理策略**:

1. **回顧**: 回到最初大綱和角色願望
2. **衝突**: 加入新的衝突或角色
3. **意外**: 引入不可預料的事件
4. **回顧角色**: 讓角色做違反性格的決定

## 五、多視角敘事模板

### 5.1 視角類型選擇

| 視角類型 | 適用場景 | 優點 | 缺點 |
|----------|----------|------|------|
| **第一人稱** | 心理剖析/情感強的故事 | 深刻內心/讀者共情 | 資訊受限 |
| **第三人稱全知** | 複雜多線/龐大世界觀 | 自由切換/上帝視角 | 缺乏緊湊感 |
| **第三人稱有限** | 多角色/需要懸念 | 保持神秘/聚焦 | 需要頻繁切換 |
| **多重視角** | 團隊故事/對稱情節 | 豐富視角/公平 | 需要謹慎管理 |

### 5.2 視角切換規則

```markdown
## 視角切換模板

### 切換時機
- [ ] 場景完全改變時
- [ ] 需要揭示另一角色視角時
- [ ] 時間段跳躍時

### 切換標記
- 使用「——」分隔視角段落
- 或使用「視角A：...」「視角B：...」
- 避免同一場景內頻繁切換

### 切換頻率
- 短篇：最多2個視角
- 中篇：最多3-4個視角
- 長篇：最多5-6個視角
```

### 5.3 多角色視角平衡

```markdown
## 視角分配原則

### 主要視角角色 (50-70% 篇幅)
- 主角：必備
- 主要對手：20-30%

### 次要視角角色 (10-20% 篇幅)
- 導師/盟友：10-15%
- 反派陣營：5-10%

### 視角覆蓋檢查
- [ ] 每卷至少出現一次主要視角？
- [ ] 反派視角是否有助於增加緊湊性？
- [ ] 是否每個重要角色至少有一次視角？
```

```markdown

| 章節 | 類型 | 節奏強度 | 目的 |
|------|------|----------|------|

### 節奏模式參考

#### 模式A: 遞進式 (適合升級流)
```
★ → ★★ → ★★★ → ★★★★ → ★★★★★ → ★★☆ → ★★★ → ★★★★★★
```

#### 模式B: 波浪式 (適合情感類)
```
★★★ → ★★★★ → ★★☆ → ★★★★ → ★★★★★ → ★★★ → ★★
```

#### 模式C: 階梯式 (適合權謀類)
```
★★ → ★★ → ★★★ → ★★★ → ★★★★ → ★★★★ → ★★★★★

### 7.2 高潮分布原則

```markdown
## 高潮設計規則

### 數量要求
- 短篇 (5章): 1-2個小高潮 + 1個大高潮
- 中篇 (15章): 3-4個小高潮 + 1-2個大高潮
- 長篇 (40章): 8-10個小高潮 + 3-4個大高潮 + 1個終極高潮

### 分布建議
- 每5-7章一個小高潮
- 每10-15章一個大高潮
- 倒數第3卷安排最終高潮
```

## 十三、劇情連貫保障系統

### 13.1 連貫性核心原則

```markdown
## 連貫性三大支柱

### 1. 時間連貫
- 每章結尾記錄「+1天」或具體時間
- 避免時間跳躍矛盾
- 特殊時間點（節日/季節）需要提前標記

### 2. 空間連貫
- 記錄角色當前位置
- 移動需要合理時間
- 避免瞬間移動（除非有特殊能力）

### 3. 邏輯連貫
- 角色決定需要有合理鋪墊
- 事件發展需要有因果關係
- 避免「突然覺醒」式發展
```

### 13.2 狀態追蹤模板

```markdown
## 全局狀態記錄（每章必備）

### 基礎狀態
- 當前時間: [Day X - 時間段]
- 當前地點: [位置]
- 當前季節: [季節]

### 角色狀態
| 角色 | 位置 | 情緒 | 狀態變化 |
|------|------|------|----------|
| 主角 | 城內 | 焦慮 | 得知反派計畫 |

### 物品/道具
- [道具名]: [狀態/位置]
- 魔法杖: 損壞中，須修復

### 關鍵資訊
- [ ] 已知道密謀內容
- [ ] 已見過反派真面目
- [ ] 還未發現背叛者
```

### 13.3 連貫性檢查清單

```markdown
## 連貫性自我檢查

### 時間檢查
- [ ] 本章時間與上一章連接自然？
- [ ] 角色有足夠時間做所述的事？
- [ ] 沒有時間矛盾？

### 空間檢查
- [ ] 角色位置變化合理？
- [ ] 移動時間估計正確？
- [ ] 沒有地點矛盾？

### 邏輯檢查
- [ ] 角色決定有足夠鋪墊？
- [ ] 衝突發展有原因？
- [ ] 轉折有預兆？

### 伏筆檢查
- [ ] 本章埋下的伏筆有記錄？
- [ ] 之前的伏筆有回收？
- [ ] 回收方式合理？
```

### 13.4 常見連貫問題與修復

| 問題 | 診斷信號 | 修復方法 |
|------|----------|----------|
| 性格突變 | 角色做出不符合性格的決定 | 回顧前3章性格表現 |
| 能力退化 | 角色突然忘記會的技能 | 加入「受傷/疲憊」理由 |
| 遺忘信息 | 角色不知道讀者已知的信息 | 加入「回憶」場景 |
| 道具消失 | 重要道具突然不見 | 加入「丟失/被偷」情節 |
| 矛盾天氣 | 夏天突然下雪 | 檢查時間線 |

## 十五、長篇質量控制

### 15.1 質量保障層級

```markdown
## 三層質量保障

### Layer 1: 即時檢查（每章生成後）
- 字數檢查
- 基本連貫性
- 格式正確

### Layer 2: 中期檢查（每10章）
- 節奏檢查（是否符合結構）
- 懸念檢查（是否有效鋪設/回收）
- 角色檢查（是否有成長/變化）

### Layer 3: 總體檢查（每卷結束）
- 主題檢查（是否扣住核心主題）
- 高潮檢查（是否達到預期強度）
- 伏筆檢查（是否全部回收）
```

### 15.2 中期質量檢查清單

```markdown
## 每10章檢查清單

### 結構檢查
- [ ] 前10章是否涵蓋「建置-對抗-解決」結構？
- [ ] 是否有至少2個明顯的高潮點？
- [ ] 角色是否至少有1次重大轉變？

### 懸念檢查
- [ ] 開頭的懸念是否已揭曉或深化？
- [ ] 中間設置的懸念是否還有效？
- [ ] 是否有為後10章準備的新懸念？

### 節奏檢查
- [ ] 前5章與後5章節奏是否不同？
- [ ] 是否有讓讀者喘息的平緩章節？
- [ ] 衝突是否逐漸升級？

### 角色檢查
- [ ] 主角是否比開始時更強/不同？
- [ ] 是否有角色離開/加入？
- [ ] 角色關係是否有變化？
```

### 15.3 質量回溯修復

```markdown
## 發現問題後的修復流程

### 問題分類
1. 輕微問題（可忽略）
   - 某句台詞不夠好
   - 某段描述可更精簡

2. 中等問題（局部修復）
   - 某章節奏太快
   - 某角色性格有偏差
   - 某懸念回收不當

3. 嚴重問題（需重寫）
   - 時間線完全混亂
   - 核心設定矛盾
   - 角色行為完全不合理

### 修復優先級
1. 先修復最嚴重的問題
2. 修復時要檢查是否影響後續章節
3. 修復後重新檢查連貫性
```

## 三十九、配角與反派設計系統

### 39.1 配角功能分類表

```markdown
## 配角設計模板

### 功能分類

#### 1. 導師型 (Mentor)
- 功能: 教導主角/提供知識/給予道具
- 登場時機: 主角需要幫助時
- 離場時機: 教會主角後
- 設計要點: 有缺陷的導師更真實

#### 2. 盟友型 (Ally)
- 功能: 陪伴/支持/提供幫助
- 登場時機: 早期建立關係
- 離場時機: 背叛/犧牲/使命完成
- 設計要點: 有自己的目標和成長

#### 3. 誘惑型 (Temptress)
- 功能: 考驗主角/增加複雜度
- 登場時機: 感情線需要時
- 設計要點: 不能只是「壞」，要有理由

#### 4. 喜劇型 (Comic Relief)
- 功能: 緩解氣氛/揭示主題
- 登場時機: 緊張情節後
- 設計要點: 適時適度，不要喧賓奪主

#### 5. 犧牲型 (Sacrifice)
- 功能: 推動主角成長/增加悲劇色彩
- 登場時機: 中期
- 設計要點: 死亡要有意義

#### 6. 間諜型 (Double Agent)
- 功能: 製造驚喜/揭示真相
- 設計要點: 提前鋪墊，不要太明顯
```

### 39.2 反派設計層次

```markdown
## 反派設計深度表

### Level 1: 工具人反派
- 特點: 只為了阻礙主角
- 問題: 單薄，沒有記憶點
- 改進: 給一個簡單的過去

### Level 2: 有理由的反派
- 特點: 有合理的動機
- 問題: 可能讓讀者同情
- 設計: 動機要與主角目標直接衝突

### Level 3: 悲劇反派
- 特點: 曾經是好人，有慘痛經歷
- 問題: 可能搶主角風頭
- 設計: 經歷要與主題相關

### Level 4: 镜像反派
- 特點: 與主角非常相似，是「另一種可能」
- 問題: 需要大量鋪墊
- 設計: 從開頭就開始暗示

### Level 5: 哲學反派
- 特點: 代表一種觀點，與主角針鋒相對
- 問題: 需要高深的思辨
- 設計: 主題要清晰
```

### 40.1 經典場景類型

```markdown
## 必備場景設計

### 1. 覺醒場景 (主角首次發現能力)
- 情緒: 震驚/恐懼/興奮
- 節奏: 突然發生 → 失控 → 意外控制
- 關鍵: 要有見證者或後果

### 2. 失敗場景 (主角首次重大失敗)
- 情緒: 絕望/不甘/反思
- 節奏: 自信 → 被打敗 → 低谷
- 關鍵: 為復仇/成長埋下種子

### 3. 失去場景 (重要的人/物離開)
- 情緒: 悲傷/崩潰/堅強
- 節奏: 幸福 → 失去 → 絕望 → 轉機
- 關鍵: 影響主角接下來的決定

### 4. 抉擇場景 (重要選擇)
- 情緒: 糾結/掙扎/決斷
- 節奏: 選項出現 → 利弊分析 → 最終選擇
- 關鍵: 選擇要體現角色成長

### 5. 對質場景 (與反派對質)
- 情緒: 憤怒/質問/震驚
- 節奏: 對峙 → 揭露真相 → 衝突升級
- 關鍵: 真相要衝撃

### 6. 犧牲場景 (自我犧牲或看見犧牲)
- 情緒: 震撼/悲痛/決然
- 節奏: 危機 → 決定 → 犧牲 → 影響
- 關鍵: 犧牲要有意義

## 四十一、Token優化技巧

### 41.1 信息密度優化

```markdown
## 高效表達技巧

### 用動作代替描述
- 不好: 「他很生氣，拳头緊緊握着」
- 更好: 「他哼了一聲，拳头砸在桌上」

### 用對話代替解釋
- 不好: 「系統顯示這是一個致命錯誤」
- 更好: 「系統：警告，致命錯誤。博士臉色一沉」

### 用氛圍代替形容詞
- 不好: 「房間裡充滿了神秘的氣息」
- 更好: 「牆上的符文發著微光，房間裡安靜得能聽見心跳」

### 用結果代替過程
- 不好: 「他練了三年的劍」
- 更好: 「他拔劍，收劍，對手已倒地」
```

### 41.2 重複信息避免

```markdown
## 避免重複

### 避免重複描述
- 第一次: 「她是個沉默寡言的人」
- 之後: 用行動表現她的沉默

### 避免重複解釋
- 設定說過一次就不用再說
- 用「正如她預料的」代替重複解釋

### 避免重複衝突
- 每章衝突要有新意
- 不要用同樣的方式解決衝突
```

### 1. 劇情崩潰
症狀: 角色行為不合理/時間線混亂/設定矛盾
原因: 沒有追蹤狀態/隨意添加設定
解決: 回溯檢查/重讀大綱

### 2. 節奏失控
症狀: 太慢/太快/沒有高潮
原因: 衝突不夠/節奏單一
解決: 增加衝突/調整章節長度

### 3. 角色走偏
症狀: 角色性格變了/目標忘了
原因: 沒有回顧角色設定
解決: 重讀角色檔案/增加角色相關情節

### 4. 寫不出來
症狀: 卡在某一章
原因: 不知道下一步/失去靈感
解決: 回顧大綱/加入新元素
```

2. 增加檢查點
3. 記錄教訓

## 四十六、緊急救援指南

### 46.1 完全卡死時

```markdown
## 卡死時的處理

### 症狀: 不知道下一章寫什麼

### 解決方案 (按順序嘗試)

1. **回顧大綱**
   - 主角當前的目標是什麼？
   - 最大的障礙是什麼？
   - 有哪些衝突還沒解決？

2. **加入新元素**
   - 加入一個新角色
   - 引入一個新信息
   - 創造一個意外事件

3. **強制轉折**
   - 讓角色做一個決定
   - 引入外部勢力
   - 打破現有平衡

4. **回到上一章**
   - 也許節奏太快
   - 也許需要更多鋪墊
   - 也許缺少過渡
```

### 46.2 質量下降時

```markdown
## 質量下降時的處理

### 症狀: 寫的內容越來越差

### 解決方案

1. **暫停生成**
   - 不要繼續寫爛的內容
   - 讓自己冷靜下來

2. **分析原因**
   - 是太累了？
   - 是沒有靈感？
   - 是設定有問題？

3. **針對性解決**
   - 累了: 休息
   - 沒靈感: 回顧大綱或加入新元素
   - 設定問題: 停下來重新設計

4. **從好的地方繼續**
   - 找到最後寫得好的地方
   - 從那裡繼續
   - 跳過寫不好的部分
```
