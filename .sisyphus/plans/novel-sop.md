## 一、 標準化流程（SOP）

### 生成前準備

#### 1.1 文件命名與位置
| 類型 | 路徑 | 範例 |
|------|------|------|
| 章節 | `chapters/ch{NN}.md` | `chapters/ch21.md` |
| 結構規劃 | `.sisyphus/plans/novel-structure-chX-Y.md` | `novel-structure-ch21-30.md` |
| 執行計劃 | `.sisyphus/plans/novel-writing-chX-Y.md` | `novel-writing-ch21-30.md` |
| 規範文件 | `.sisyphus/plans/novel-spec.md` | - |
| 流程手冊 | `.sisyphus/plans/novel-sop.md` | - |

#### 1.2 生成前檢查清單
- [ ] 讀取上一章結尾的全局記憶備忘錄
- [ ] 確認 current asset status
- [ ] 確認 current suspense/plot threads
- [ ] 確認 character statuses
- [ ] 檢查 novel-spec.md 是否為最新版本
- [ ] 確認寫作比例（5%技術 + 30%場景 + 35%心理 + 30%對話）
- [ ] 參考結構規劃文件確認本章節情節

### 生成流程（按順序執行）

#### Step 1: 讀取上一章結尾的全局記憶備忘錄
- 確認當前時間線
- 確認資產狀態
- 確認懸念和伏筆
- 確認登場人物狀態

#### Step 2: 檢查並更新規範文件
- 確認規範版本為最新 (novel-spec.md)
- 確認技術比例為 5%
- 確認資產數據準確

#### Step 3: 創建章節大綱計劃（必要時）
- 如果是新一卷（每10章），創建詳細大綱
- 明確每章的核心情節
- 明確懸念安排

#### Step 4: 生成單一章節
按照以下模板結構生成:

```markdown
# 第 N 章：[標題]

[第一段：場景描寫 + 與上一章懸念銜接]
- 描述當前場景
- 銜接上一章的結尾

[主體段落：按比例分配]
- 技術氛圍（一句話）
- 場景描寫
- 心理獨白
- 對話與動作

[結尾段落：懸念]
- 為下一章做鋪墊

---
> **全局記憶備忘錄更新：**
> * 當前時間線：Day N — [時間段]
> * 主角資產狀態：[BTC、功德等]
> * 重要伏筆與未解之謎：
>   - 懸念1
>   - 懸念2
> * 登場人物/神明狀態：[狀態]
```

#### Step 5: 驗證與提交
- [ ] 檢查資產變動是否合理
- [ ] 檢查懸念是否延續
- [ ] 檢查字數是否接近 2000
- [ ] 檢查技術比例是否為 5%
- [ ] Git commit and push

## 二、 章節模板詳解

### 2.1 文件頭部格式
```markdown

```

### 2.2 內容結構與比例
| 元素 | 比例 | 字數參考 |
|------|------|----------|
| 技術氛圍 | 5% | ~100字（一句話帶過） |
| 場景感官 | 30% | ~600字（環境、氛圍、感官） |
| 心理獨白 | 35% | ~700字（內心掙扎、PM/量化思維） |
| 對話動作 | 30% | ~600字（對話交鋒、表情） |
| **總計** | 100% | **~2000字** |

```markdown
> * 重要伏筆與未解之謎：
>   - [懸念1]
>   - [懸念2]

```
