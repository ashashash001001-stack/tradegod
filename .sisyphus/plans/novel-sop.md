# 《量化神明》章節生成 - 標準化流程手冊

> **版本**: v1.0
> **建立日期**: 2026-04-06
> **適用範圍**: 所有章節生成任務

---

## 一、 標準化流程（SOP）

### Step 1: 讀取上一章結尾的全局記憶備忘錄
- 確認當前時間線
- 確認資產狀態
- 確認懸念和伏筆
- 確認登場人物狀態

### Step 2: 檢查並更新規範文件
- 確認規範版本為最新 (novel-spec.md)
- 確認技術比例為 5%
- 確認資產數據準確

### Step 3: 創建章節大綱計劃（必要時）
- 如果是新一卷（每10章），創建詳細大綱
- 明確每章的核心情節
- 明確懸念安排

### Step 4: 生成單一章節
- 按照 5%技術 + 30%場景 + 35%心理 + 30%對話 的比例
- 融入全局記憶備忘錄中的劇情
- 章節結尾必須包含「全局記憶備忘錄更新」

### Step 5: 驗證與提交
- 檢查資產變動是否合理
- 檢查懸念是否延續
- Git commit and push

## 五、 Git 提交規範

### 提交訊息格式
```
[類型]: [簡短描述]

- 詳細內容1
- 詳細內容2
- 詳細內容3

Ultraworked with [Sisyphus]
Co-authored-by: Sisyphus <clio-agent@sisyphuslabs.ai>
```

### 提交類型
- `feat`: 新功能/新情節
- `fix`: 修復問題
- `docs`: 文檔更新
- `chore`: 其他更新
